# Privacy and Sovereignty

SELF implementations should preserve user sovereignty by default.

## Principles

- Collect only what is necessary.
- Explain what each data source is used for.
- Separate verified data from inference.
- Provide deletion and export paths.
- Avoid dependency loops.
- Do not diagnose users.
- Do not present symbolic interpretation as verified reality.

## Data Boundary

A SELF agent should not access journals, messages, calendars, purchases, location patterns, or platform activity unless the user has explicitly consented.
