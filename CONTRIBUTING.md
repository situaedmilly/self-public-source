# Contributing

Contributions should strengthen precision, evidence classification, sovereignty, and implementation clarity.

## Accepted Contributions

- prompt refinements
- architecture notes
- response templates
- privacy improvements
- safety boundaries
- implementation examples

## Rejected Contributions

- diagnostic language
- coercive behavioral design
- dependency-forming interaction loops
- claims that blur evidence with speculation
- spiritual or symbolic language presented as factual authority

## Standard

Every contribution should make SELF more precise, sovereign, evidence-oriented, and executable.
