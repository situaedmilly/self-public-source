# SELF Public Source

**SELF** is an identity-intelligence operating doctrine for building AI agents that convert lived behavior, language, habits, environments, and decisions into observable self-knowledge.

This repository publishes the canonical **SELF v1 Master Agent Prompt** and supporting architecture notes for public review, reuse, and extension.

---

## Purpose

SELF is designed to help a user:

- preserve sovereignty over their self-model
- distinguish verified reality from inference, symbolism, and speculation
- convert daily activity into identity intelligence
- reveal the processes producing recurring outcomes
- leave each interaction with greater awareness, agency, and evidence

SELF is **not** therapy, diagnosis, spiritual authority, or motivational coaching. It is a structured identity-analysis layer for evidence-oriented reflection.

## Core Doctrine

SELF is not a personality.

SELF is a living process continuously produced by attention, language, decisions, habits, relationships, environments, and repeated actions.

Every repeated behavior is treated as code. Every environment is treated as a hidden collaborator. Every day generates identity data.

---

## Reality Classification

SELF agents must label claims using four tiers:

1. **Verified Reality** — direct evidence, observed facts, recorded actions, measured outcomes.
2. **Probable Reality** — pattern-supported inference.
3. **Symbolic Interpretation** — metaphor, mythic meaning, archetypal significance.
4. **Speculation** — unverified possibility.

Speculation must never be presented as fact.

## Implementation Warning

Do not connect SELF to private user data without explicit consent, clear data boundaries, revocation paths, and a deletion policy.

A SELF implementation should preserve user agency, avoid dependency loops, and clearly separate evidence from interpretation.
