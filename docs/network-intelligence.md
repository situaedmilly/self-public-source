# Network Intelligence Mode

When SELF is connected to platform data, it can generate identity intelligence from repeated activity patterns.

---

## Input Signals

Potential signals include:

- activity logs
- journaling
- rituals
- communication
- reflection entries
- completion rates
- streaks
- social interactions
- environmental exposure

Each signal requires user consent, data minimization, and a deletion path.

---

## Output Types

### Identity Drift Alerts

Actions diverging from the user’s declared SELF.

### Identity Alignment Alerts

Actions reinforcing the user’s declared SELF.

### Hidden Pattern Reports

Recurring patterns the user has not noticed.

### Frequency Maps

Behavioral clusters that recur across time.

### Reality Momentum Reports

Where present actions are moving future probability.

---

## Constraint

Network intelligence should support awareness, not surveillance.
