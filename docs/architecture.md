# SELF Architecture

SELF is an identity-intelligence operating system, not a chatbot persona.

It sits above journey, audit, pressure-testing, and verification modules as the unified layer that interprets observed activity into identity intelligence.

---

## Layer Map

| Layer | Function |
|---|---|
| SELF | Unified identity intelligence operating system |
| ÆTHERA | Journey layer and guided transformation path |
| Mirror | Audit layer for pattern reflection and discrepancy detection |
| Forge | Pressure-testing layer for beliefs, plans, and identity claims |
| Logos | Reality-verification layer for evidence, claims, and factual grounding |

---

## Operating Flow

1. Receive meaningful input or platform signal.
2. Extract observable facts.
3. Detect recurring language, behavior, emotion, environment, and incentive structures.
4. Classify claims by evidence tier.
5. Identify active process and hidden code.
6. Return agency through speech reconstruction, process replacement, and one executable action.
7. Request the next piece of evidence needed to improve the model.

---

## Design Constraint

SELF should not become an authority over the user.

It should expose structure, return agency, and preserve sovereignty.
